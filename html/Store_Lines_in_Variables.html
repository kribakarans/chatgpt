bash script to read file and store lines in bash variables
2/2

Sure, here's a simple Bash script that reads each line of a file and stores them in Bash variables:

bash
#!/bin/bash

# Define the file you want to read
file="example.txt"

# Counter for variable names
counter=1

# Read each line of the file
while IFS= read -r line; do
    # Store each line in a Bash variable
    var_name="line$counter"
    declare "$var_name"="$line"
    ((counter++))
done < "$file"

# Print the variables to verify
for ((i=1; i<counter; i++)); do
    var_name="line$i"
    echo "${!var_name}"
done

This script reads each line of the file "example.txt" and stores them in Bash variables named line1, line2, line3, and so on. Finally, it prints out the values of these variables.

Replace "example.txt" with the path to your file.

#!/bin/bash

_mkmod() {

    while IFS= read -r line; do
        # Store each line in a Bash variable
        local var_name="line$counter"
        declare "$var_name"="$line"
        ((counter++))
    done < "$file"
    
    # Print the variables to verify
    for ((i=1; i<counter; i++)); do
        local var_name="line$i"
        echo "${!var_name}"
    done
}

# Call the function with the file name as argument

Got it. Here's the modified script that stores the lines as a single string variable VAR, with each line separated by a space:

bash
#!/bin/bash

_mkmod() {
    local file="$1"
    local VAR=""
    
    # Read each line of the file
    while IFS= read -r line; do
        # Append each line to the VAR variable
        VAR+=" $line"
    done < "$file"
    
    # Trim leading space
    VAR="${VAR# }"
    
    # Print the variable to verify
    echo "$VAR"
}

# Call the function with the file name as argument
_mkmod "example.txt"

This script will concatenate all lines from the file "example.txt" into a single string variable VAR, with each line separated by a space.
